Spec preview forge/feat-auth-rotate

Rotate OAuth refresh tokens every 24h

feat(auth): rotate oauth refresh tokens

Context

The current OAuth implementation in src/auth/refresh.ts issues a single long-lived refresh token at login and never rotates it. Compliance flagged this on the Q1 audit because a leaked token is valid for the full 90-day session window.

Acceptance criteria

  • Refresh tokens rotate every 24h, with grace period of 30 minutes for in-flight requests.
  • Old token is revoked after grace period and a revocation entry is written to auth_revocations.
  • Token rotation is observable via the auth.refresh.rotated metric.
  • No regression in the tests/auth/refresh.test.ts suite.

Quality gates

  • bun test tests/auth
  • bun run lint
  • bunx tsc --noEmit
